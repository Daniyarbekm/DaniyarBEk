Git и GitHub Flow

1. Что такое GitHub Flow?

GitHub Flow — это легкий процесс ветвления, предназначенный для эффективной командной работы.

Он используется для управления разработкой, где каждая новая задача или фича реализуется в отдельной ветке, которая в итоге сливается с основной (обычно main).

2. Что такое Git Rebase и Git Merge?

Git Rebase: Переписывает историю коммитов, добавляя изменения из одной ветки в другую последовательно.

Git Merge: Объединяет изменения из разных веток, создавая новый коммит, который включает историю обоих веток.

3. Git Fetch и Git Pull

Git Fetch: Загружает новые данные из удаленного репозитория, но не сливает их с текущей веткой.

Git Pull: Одновременно выполняет fetch и merge, обновляя локальный репозиторий новыми изменениями из удаленного.

4. Как использовать Git Fetch на GitHub?

Выполни команду git fetch, чтобы скачать обновления с удаленного репозитория, не изменяя свой локальный код. Это полезно, если нужно проверить обновления перед слиянием их вручную.
