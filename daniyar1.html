Git и GitHub Flow

1. Что такое Git WorkFlow?

Git Workflow — это, по сути, набор шагов и правил, по которым разработчики взаимодействуют с Git. Он помогает организовать процесс разработки, управление ветками и слияния.

2. Git Rebase и Git Merge — в чём разница?

Git Rebase: Эта штука переписывает историю коммитов. Ты как бы переносишь все свои изменения поверх другой ветки.

Git Merge: А тут просто убираешь две ветки. История будет ветвиться, но зато ничего не теряется и всё сохраняется как есть.

3. Git Fetch и Git Pull — что к чему?

Git Fetch: Эта команда просто скачивает обновления с удалённого репозитория, но не убирает их с твоими файлами. Типа проверяешь, что нового, но ничего не меняешь.

Git Pull: А вот тут fetch и merge одновременно — всё новое сразу в твоей ветке.

4. Как пользоваться Git Fetch на GitHub?

Пиши git fetch, если хочешь просто скачать обновления с удалённого репозитория, но не трогать свой код. Это полезно, если хочешь сначала посмотреть, что изменилось, а потом решать, убирать или нет.
